## 附章：RAG-Anything 独有能力（火山引擎矩阵未覆盖）

以下功能为 RAG-Anything 已实现但火山引擎产品矩阵中**没有对应行**的能力：

### 架构自由度

| # | RAG-Anything 独有 | 说明 | 优势 |
|---|------------------|------|------|
| 1 | **模型无关架构** | 支持通义千问/DeepSeek/MiniMax/Ollama/LMStudio/OpenAI 等 7+ 供应商，切换模型只改 1 个环境变量 | 避免供应商锁定，甲方可复用已采购模型 |
| 2 | **本地模型离线运行** | Ollama / LMStudio 本地模型即插即用，零 API 费用，完全离线 | 涉密场景数据不出域 |
| 3 | **源码交付** | 完整 Python/React 源码 + 架构文档 + 二次开发培训 | 甲方完全掌控，可自主定制 |

### 智能体体系

| # | RAG-Anything 独有 | 说明 |
|---|------------------|------|
| 4 | **5 种预置智能体模板** | 通用文档📋 / 学术论文🎓 / 代码文档💻 / 精确检索🔍 / 创意头脑风暴🧠，每模板独立配置 temperature/mode/top_k/prompt |
| 5 | **5 种查询检索模式** | hybrid(混合) / local(本地) / global(全局) / naive(朴素) / mix(混合模式)，按场景精确切换 |
| 6 | **Agent 级 Rerank 开关** | 每个智能体独立配置 `enable_rerank`，按需控制成本 |

### 工程品质

| # | RAG-Anything 独有 | 说明 |
|---|------------------|------|
| 7 | **OMML 数学公式提取器（自研）** | Word 内嵌公式 → LaTeX 自动转换，保留结构化数学内容 |
| 8 | **子进程 Worker 隔离** | 文档处理在独立进程运行，超时 300s 自动 kill，不阻塞主服务 |
| 9 | **Access + Refresh 双 Token** | JWT HS256 + 独立 Refresh 密钥，24h + 7d 双有效期，支持无感刷新 |
| 10 | **本地 OCR（PaddleOCR）** | 即开即用，零 API 费用，无需联网，支持中英混合识别 |

### 运维与部署

| # | RAG-Anything 独有 | 说明 |
|---|------------------|------|
| 11 | **Docker 四服务编排** | app + PostgreSQL + Redis + Nginx，docker-compose up -d 一键启动 |
| 12 | **四级健康检查** | 应用(30s) + 数据库(10s) + Redis + Nginx，自动重启 |
| 13 | **敏感日志自动脱敏** | mask_sensitive_data() 自动脱敏 password/token/secret/key/api_key |
| 14 | **Brute Force 账号锁定** | 账号级别：5次登录失败锁定15分钟（MAX_FAILED_LOGIN_ATTEMPTS=5） |

### 火山有但 RAG-Anything 没有的

| 火山独有 | 说明 |
|---------|------|
| 30+ 数据库/数仓接入 | RAG-Anything 目前 2 种 (MySQL + PostgreSQL) |
| 数据集模块（Clickhouse/Hive 管理） | RAG-Anything 未实现 |
| 非结构化数据打标应用（7 项） | 独立的 AI 打标产品线，RAG-Anything 未覆盖 |
| 豆包全家桶一步集成 | 火山生态内 ASR/OCR/VLM 免配置直接调用 |

---

> **综合结论**: 火山引擎覆盖广度占优（102 项功能矩阵），RAG-Anything 在**架构自由度、智能体体系、工程品质、运维透明度**四个维度有火山矩阵未覆盖的独特优势。两套系统的关系是互补而非替代——火山适合需要"开箱即用全家桶"的客户，RAG-Anything 适合需要"源码掌控+模型自由+成本可控"的客户。
